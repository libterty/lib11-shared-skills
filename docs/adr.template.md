# Architecture Decision Record — 範本

**這是範本，不是拿來直接編輯的活文件。** 每個決策獨立一個檔案：複製這份範本到 `docs/adr/<流水號>-<決策簡述>.md`（例如 `docs/adr/0001-message-queue-choice.md`），一個決策一個檔案，不要把多個決策塞進同一份裡。`docs/adr/` 底下的檔案預設被 `.gitignore` 排除，不會被提交，因為裡面通常會累積真實的系統名稱/架構細節等內部資訊。

用 `shared-skills/architecture-decision-record/SKILL.md` 產出內容後，貼進複製出來的檔案裡存查；之後要調整（例如狀態從「提議中」變成「已決定」、或要記錄後續的 rollback）也是直接編輯那份檔案。

欄位對應 `architecture-decision-record/SKILL.md` 的 Output Contract，順序固定，缺一不可：

---
