# Meeting Notes（單次會議原始紀錄）— 範本

**這是範本，不是拿來直接編輯的活文件。** 複製這份檔案到 `docs/mn/<日期>-<會議簡稱>.md`（例如 `docs/mn/2026-07-15-weekly.md`），每次開會存一份新檔案，不要覆蓋舊的——`docs/mn/` 底下的檔案預設被 `.gitignore` 排除，不會被提交，因為裡面通常會累積真實的逐字稿/決策/人名等內部資訊。

## 這份檔案存在的原因

`docs/ar.template.md`、`docs/raci.template.md` 的「來源」欄位會寫「哪次會議/場合產生的（日期＋會議名稱）」，但那只是一段文字描述，不代表背後真的有一份可以回頭查證的原始記錄。如果你希望「來源」欄位能真的連到一份存檔（而不是事後想不起來這次到底討論了什麼），就把每次會議的原始逐字稿/筆記存一份在 `docs/mn/` 底下，讓 `docs/ar/<檔名>.md`、`docs/raci/<檔名>.md` 的「來源」欄位可以直接引用這份檔案的路徑。

如果你不需要逐次回溯、只想要一句話描述來源，可以不用建立這份檔案，「來源」欄位照樣可以只寫「2026-07-15 週會」這種自由文字——這份範本是選用的，不是強制流程。
