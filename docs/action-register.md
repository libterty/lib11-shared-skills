# Action Register（行動追蹤表）

你直接管控、手動編輯的活文件。每次開完會，把新的 action item 加進「進行中」表格；狀態改變時直接改那一列；完成的移到「已完成」區塊存查，不要刪除（保留追蹤歷史）。

如果嫌手動更新麻煩，可以把會議逐字稿/筆記貼給 Claude，套用 `shared-skills/action-register-maintainer/SKILL.md`，讓它幫你比對這份表格、提出「新增/更新/標記逾期」的建議，但**最終要不要採用、要不要真的改這份檔案，由你決定**——那個 skill 只產出建議 diff，不會自己動手改這份檔案。

## 欄位說明

| 欄位 | 說明 |
|---|---|
| ID | 流水號，方便引用（例如 `AR-001`） |
| 項目 | 具體的行動內容 |
| 負責人 | 誰要做 |
| 來源 | 哪次會議/場合產生的（日期＋會議名稱） |
| 建立日期 | 第一次被記錄下來的日期 |
| 截止日期 | 預計完成日期，沒有就寫「未定」 |
| 狀態 | 進行中／卡住／待確認／已完成 |
| 最後更新 | 這一列最後一次被異動的日期 |
| 備註 | 卡住的原因、依賴、其他上下文 |
